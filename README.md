# 1. Описание предметной области
Приложение предназначено для управления каталогом мотоциклов в салоне. Позволяет добавлять технику, сортировать ее по объему двигателя, искать по ключевым словам и выгружать аналитические срезы в фоновом режиме.

# 2. Побайтовая спецификация формата `data.bin`
| Смещение | Поле | Тип данных | Размер (в байтах) | Описание кодирования (Спецификация) |
| :--- | :--- | :--- | :--- | :--- |
| 0 | id | int32 | 4 | Уникальный ID мотоцикла в порядке Big-Endian |
| 4 | brandLen | int32 | 4 | Длина строки названия бренда в байтах (Big-Endian) |
| 8 | brand | UTF-8 String | N (равно brandLen) | Текстовое название модели (кодировка UTF-8) |
| 8 + N | type | byte | 1 | Порядковый индекс (ordinal index) из перечисления MotorcycleType |
| 9 + N | engineVolume | float64 | 8 | Объем двигателя (дробное число IEEE 754, Big-Endian) |
| 17 + N | isAvailable | byte | 1 | Логический флаг наличия: 0x00 = false, 0x01 = true |
| 18 + N | commentFlag | byte | 1 | Флаг для nullable: 0x00 — null, 0x01 — данные присутствуют |
| 19 + N (опц.)| commentLen | int32 | 4 | Длина строки комментария (пишется только при commentFlag = 0x01) |
| 23 + N (опц.)| comment | UTF-8 String | M (равно commentLen)| Текст комментария (пишется только при commentFlag = 0x01) |


# 3. Архитектура

    services/

    ui/

 logs.txt                      # Системный лог работы приложения 

 # 3. Сборка и Запуск
Для запуска проекта необходимо открыть терминал и прописать `dart run bin/main.dart`.
Если в системе отсутствует файл data.bin, ядро перехватит FileSystemException. Приложение запишет системное уведомление в лог , развернет пустую коллекцию и сгенерирует 5 демонстрационных тестовых объектов для проверки работы CRUD без ручного ввода.
